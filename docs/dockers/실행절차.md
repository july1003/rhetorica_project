### 실행 순서

#### Step 1: 파일 위치 확인 및 터미널 이동

준비하신 4개의 파일(`docker-compose.yml`, `Dockerfile`, `Dockerfile.batch`, `requirements.txt`)을 하나의 폴더(예: `my-project`)에 모두 모아주세요. 그 후 해당 폴더에서 터미널(또는 CMD/PowerShell)을 엽니다.

#### Step 2: Docker 이미지 빌드 (Build)

설정 파일들을 바탕으로 실행 가능한 이미지를 만듭니다. `requirements.txt`에 적힌 패키지들이 이 단계에서 설치됩니다.

* **명령어:** docker-compose build --no-cache
* **설명:** `Dockerfile`과 `Dockerfile.batch`를 참조하여 각각의 서비스 이미지를 생성합니다.

C:\big20\final\docs\dockers\C_docker>docker-compose build --no-cache

#### Step 3: 컨테이너 실행 (Up)

빌드된 이미지를 바탕으로 실제 컨테이너를 띄웁니다.

* **명령어:** `docker-compose up -d`
* **설명:** `-d` 옵션은 '데몬(Background)' 모드로, 터미널을 닫아도 컨테이너가 계속 돌아가게 합니다.

#### Step 4: 실행 상태 확인

컨테이너들이 정상적으로 구동 중인지 확인합니다.

* **명령어:** `docker-compose ps`
* **명령어**: docker compose images
* **설명:** `State`가 `Up`으로 표시되면 성공입니다. 만약 오류가 난다면 `docker-compose logs`를 입력해 원인을 파악할 수 있습니다.
