- **특징:** 구직신청, 실업급여, 직업훈련(내일배움카드) 등 고용 관련 서비스를 통합 제공합니다.
- **장점:** 정부가 인증한 구인 정보가 많아 신뢰도가 높고, 적성검사나 진로 상담 서비스도 무료로 이용할 수 있습니다.

- **장점:** 기관별 연봉 정보, 복리후생, 채용 통계 등 상세한 공공기관 정보를 비교 분석할 수 있습니다.

- **특징:** 서울교통공사, 각 지역 도시공사 등 지방자치단체 산하 공공기관 채용 정보를 집중적으로 제공합니다.

공공데이터에서 api 를 제공하지 않나?





네, 맞습니다. 대한민국 정부는 **'공공데이터포털'**을 통해 채용 정보를 포함한 방대한 데이터를 API 형태로 제공하고 있습니다.

개발자나 데이터 분석가라면 웹사이트 화면을 보는 것보다 API를 통해 데이터를 직접 받아오는 것이 훨씬 효율적일 것입니다.

------

### 1. 공공데이터포털 (data.go.kr)

가장 핵심적인 창구입니다. 여기서 **'채용'**, **'일자리'**, **'공공기관 채용'** 등의 키워드로 검색하면 다양한 API를 찾을 수 있습니다.

- **제공 형식:** REST API (JSON/XML)
- **대표적인 채용 관련 API:**
  - **한국고용정보원_워크넷 채용정보:** 민간과 공공을 아우르는 가장 방대한 실시간 채용 공고 데이터입니다.
  - **기획재정부_공공기관 채용정보(잡알리오):** 전국의 공공기관 신입/경력 채용 공고만 타겟팅하여 가져올 수 있습니다.
  - **공무원 채용 정보:** 인사혁신처에서 제공하는 공무원 시험 및 공고 데이터입니다.

### 2. 활용 방법 (워크넷 API 예시)

워크넷 API를 사용하려면 다음과 같은 절차를 거칩니다.

1. **공공데이터포털 접속:** [data.go.kr](https://www.data.go.kr/)에서 회원가입.
2. **API 활용 신청:** 원하는 채용 데이터셋을 찾아 '활용신청' 클릭 (보통 즉시 승인).
3. **인증키 발급:** 마이페이지에서 `ServiceKey` 확인.
4. **호출:** 아래와 같은 구조의 URL로 요청을 보냅니다.

> **API 호출 구조 예시:** `http://openapi.work.go.kr/opi/opi/opia/wantedApi.do?authKey=[인증키]&callTp=L&returnType=XML&startPage=1&display=10`

### 3. API 사용 시 장점

- **자동화:** 특정 키워드(예: "데이터 분석", "서울 지역")가 포함된 공고만 필터링해서 슬랙(Slack)이나 이메일로 알림을 보낼 수 있습니다.
- **커스텀 대시보드:** 여러 사이트를 방문할 필요 없이 내가 원하는 형태의 UI로 채용 정보를 시각화할 수 있습니다.
- **트렌드 분석:** 기간별 채용 공고 수의 변화나 직무별 연봉 추이 등을 분석하기 용이합니다.

------

### ⚠️ 주의사항

- **트래픽 제한:** API마다 하루 호출 횟수 제한(예: 일 1,000회~10,000회)이 있으므로 확인이 필요합니다.
- **데이터 갱신 주기:** 실시간 반영이 되는 API도 있지만, 하루 1~2회 배치(Batch) 형태로 업데이트되는 경우도 있으니 '데이터 업데이트 주기' 항목을 체크하세요.

**혹시 특정 언어(Python, JavaScript 등)로 API를 호출하는 예제 코드가 필요하신가요?** 원하시는 언어를 말씀해 주시면 바로 작성해 드릴 수 있습니다.





공공데이터에서 채용정보를 가져올 수 있는 API 찾아줘





공공데이터포털([data.go.kr](https://www.data.go.kr/))에서 활용할 수 있는 주요 채용 정보 API를 정리해 드립니다. 목적에 따라 가장 적합한 API를 선택해 사용해 보세요.

------

### 1. 워크넷(Worknet) 채용 정보 API

가장 방대한 양의 데이터를 제공하며, 민간 기업과 공공기관 채용 정보를 모두 포함합니다.

- **제공 기관:** 한국고용정보원
- **주요 API 명칭:** `한국고용정보원_워크넷 채용정보`
- **특징:** 키워드, 직종, 지역별 상세 검색이 가능하며, 구인 상세 정보뿐만 아니라 기업 정보도 함께 연동됩니다.
- **활용처:** 일반 기업 채용 앱 개발, 구직 서비스 구축 등

### 2. 잡알리오(JOB-ALIO) 공공기관 채용 API

전국 모든 **공공기관(공기업, 준정부기관 등)**의 채용 공고만 전문적으로 가져올 수 있습니다.

- **제공 기관:** 기획재정부
- **주요 API 명칭:** `기획재정부_공공기관 채용정보 조회서비스`
- **특징:** 기관유형(시장형 공기업 등), 고용형태(정규직, 인턴 등)에 따른 정밀 필터링이 가능합니다.
- **활용처:** 공공기관 취업 특화 대시보드 제작

### 3. 클린아이(Cleaneye) 지방공공기관 API

서울교통공사와 같은 **지방 공기업 및 지자체 산하 기관**의 정보를 제공합니다.

- **제공 기관:** 행정안전부
- **주요 API 명칭:** `행정안전부_클린아이_채용공고` (또는 관련 파일 데이터)
- **특징:** 전국 지자체별 산하기관의 채용 소식을 수집하기에 적합합니다.

### 4. 인사혁신처 공공취업정보 API

**국가직 공무원 및 공공 부문**의 전문적인 취업 정보를 제공합니다.

- **제공 기관:** 인사혁신처
- **주요 API 명칭:** `인사혁신처_공공취업정보 조회`
- **특징:** 직급별, 기관별 채용 목록을 조회할 수 있습니다.

------

### 🛠️ API 활용 팁 (Python 예시)

대부분의 API는 활용 신청 후 발급받은 `ServiceKey`를 사용하여 호출합니다.

Python

```
import requests

# 예시: 워크넷 API 호출 구조
url = 'http://openapi.work.go.kr/opi/opi/opia/wantedApi.do'
params = {
    'authKey': 'YOUR_SERVICE_KEY', # 본인의 인증키
    'callTp': 'L',                # 목록 요청
    'returnType': 'JSON',         # 데이터 형식
    'startPage': 1,
    'display': 10
}

response = requests.get(url, params=params)
data = response.json()
print(data)
```

**가장 먼저 테스트해보고 싶은 특정 기관이나 직종이 있으신가요?** 말씀해 주시면 해당 조건에 맞는 API 호출 파라미터 설정을 도와드릴 수 있습니다.





“AI 기반 모의 면접 시스템” 를 만드는 프로젝트를 하려고 해.

LLM·LangChain·RAG 조합을 전제로 실무 기준에서 가장 안정적인 선택을 할 수 있는 PostgreSQL 버전을 알려줘. 15.X 생각했는데 누구는 18버전이 최선이라고 하는 사람도 있어.





AI 기반 모의 면접 시스템은 **대규모 텍스트 데이터(면접 질문/답변), 벡터 데이터(RAG용 임베딩), 그리고 대화 이력(Chat History)**을 동시에 효율적으로 관리해야 합니다.

결론부터 말씀드리면, 2026년 현재 실무 기준으로 가장 추천하는 버전은 **PostgreSQL 18**입니다. 15버전은 이미 '구형'에 가깝고, 18버전이 AI 워크로드에 최적화된 기능을 대거 포함하고 있기 때문입니다.

------

### 1. 왜 PostgreSQL 18인가? (실무적 이유)

누군가 18버전을 추천했다면, 그분은 아마 **AI 및 벡터 검색 최적화** 때문이었을 겁니다. PostgreSQL 18은 2025년 하반기에 출시되어 현재 가장 진보된 안정화 버전입니다.

| 기능          | PostgreSQL 15 (2022)    | PostgreSQL 18 (2026)          | AI 프로젝트에서의 의미                                      |
| ------------- | ----------------------- | ----------------------------- | ----------------------------------------------------------- |
| **벡터 엔진** | pgvector 기본 설치 필요 | **pgvector 0.8+ 최적화**      | RAG 속도 및 정확도 대폭 향상                                |
| **I/O 방식**  | 동기식 (Blocking)       | **비동기 I/O (Asynchronous)** | 대량의 면접 데이터 로딩/조회 시 병목 제거                   |
| **JSON 처리** | 기본적인 JSONB 지원     | **SQL:2023 JSON 스탠다드**    | LLM의 JSON 출력을 파싱하고 쿼리하기 훨씬 수월함             |
| **ID 생성**   | UUIDv4 위주             | **UUIDv7 네이티브 지원**      | 시간 순 정렬이 중요한 대화 로그(Chat History) 저장에 최적화 |

### 2. LangChain & RAG 조합에서의 이점

- **pgvector와의 궁합:** `langchain-postgres` 라이브러리는 최신 버전의 pgvector와 PostgreSQL 기능을 적극 활용합니다. 18버전은 벡터 데이터의 인덱싱(HNSW 등) 속도가 이전 버전보다 월등히 빠릅니다.
- **Virtual Generated Columns:** 면접 답변 데이터에서 특정 키워드를 추출하거나, 실시간으로 점수를 계산하여 가상 컬럼으로 만들 때 디스크 용량을 차지하지 않고 쿼리 시점에 계산할 수 있어 관리가 편합니다.

### 3. "AI 모의 면접 시스템"을 위한 아키텍처 추천

모의 면접 시스템은 다음과 같은 구조로 데이터를 저장하게 될 텐데, 이때 PostgreSQL 18의 기능이 빛을 발합니다.

1. **Vector Store (RAG):** 면접자의 답변과 모범 답안을 비교하기 위한 임베딩 데이터 저장. (pgvector 사용)
2. **Chat Message History:** LangChain의 `PostgresChatMessageHistory`를 사용해 면접 대화 흐름 저장. (UUIDv7로 시간순 정렬 최적화)
3. **JSON Metadata:** 면접 평가 결과(발음 점수, 키워드 포함 여부 등)를 JSONB 타입으로 저장하여 유연하게 관리.

------

### 💡 최종 제안

- **신규 프로젝트라면:** 주저 말고 **PostgreSQL 18**로 시작하세요. 클라우드 서비스(AWS RDS, Google Cloud SQL, Azure 등)에서도 2026년 현재 대부분 18버전을 기본(Default) 또는 안정 버전으로 지원하고 있습니다.
- **인프라 제약이 있다면:** 최소한 **17버전 이상**을 권장합니다. 15버전은 AI 프로젝트에서 요구하는 최신 인덱싱 및 JSON 처리 성능에서 차이가 큽니다.

**프로젝트의 첫 단추로 데이터베이스 스키마 설계나 LangChain 연동 코드가 필요하신가요?** 원하시면 간단한 테이블 구조와 연결 예시를 짜드릴 수 있습니다.

질문 주신 **PostgreSQL 버전 선택**에 대해, *LLM·LangChain·RAG* 조합을 전제로 **실무 기준에서 가장 안정적인 선택**을 정리해 드리겠습니다.

## 1. 결론부터: 가장 권장되는 PostgreSQL 버전

------

## 2. 왜 PostgreSQL 15가 최적인가

### 2.1 LLM / RAG 프로젝트 관점

AI 모의 면접 시스템에서는 PostgreSQL을 단순 DB가 아니라 다음 용도로 사용하게 됩니다.

- 사용자 정보 / 면접 세션 저장
- 질문–답변 로그 (대화 히스토리)
- 문서 기반 RAG 인덱스
- **Vector Search (pgvector)**

Postgres 15는 이 모든 요구사항에서 **문제 없이 검증된 버전**입니다.

------

### 2.2 pgvector 호환성 (핵심)

LangChain + RAG에서 거의 필수인 `pgvector` 기준으로 보면:

| 항목                | PostgreSQL 14 | PostgreSQL 15   | PostgreSQL 16  |
| ------------------- | ------------- | --------------- | -------------- |
| pgvector 안정성     | 안정          | **가장 검증됨** | 일부 기능 최신 |
| LangChain 기본 예제 | 지원          | **권장**        | 일부 미검증    |
| Supabase / RDS 호환 | OK            | **BEST**        | 제한적         |
| 운영 리스크         | 낮음          | **매우 낮음**   | 상대적         |

> 📌 **15.x는 pgvector + LangChain 예제가 가장 많고, 장애 사례가 적음**

------

### 2.3 PostgreSQL 16은 왜 추천하지 않는가?

- PostgreSQL 16은 **성능과 SQL 기능은 우수**
- 그러나:
  - 일부 Managed DB (RDS, Cloud SQL) 지원이 늦음
  - LangChain / vector DB 예제가 상대적으로 적음
  - 수업/과제/평가 환경에서 **“최신이라서 문제”** 되는 경우 발생

📌 **학습·프로젝트·제출용**으로는 과도하게 최신일 필요 없음

------

## 3. 프로젝트 구조 기준 추천 스택

### 3.1 DB 스택 (권장)

```text
PostgreSQL 15.x
└── pgvector 0.5.x 이상
```

- Vector dimension: OpenAI embedding (1536 or 3072)
- cosine / inner product 모두 안정

------

### 3.2 LangChain 연동 예시 (Python)

```python
from langchain.vectorstores import PGVector
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()

db = PGVector(
    connection_string="postgresql+psycopg2://user:pw@localhost:5432/interview_db",
    embedding_function=embeddings,
    collection_name="interview_docs"
)
```

➡ PostgreSQL 15에서 가장 많이 쓰이는 구성

## 4. 운영/평가 환경에서의 안전성

### 교수/멘토 관점에서 보는 안정 기준

- “특정 버전 때문에 안 된다”는 변명 방지
- 설치 문서 / 레퍼런스 충분
- Docker로 쉽게 재현 가능

Postgres 15는:

- Docker 공식 이미지 안정
- AWS RDS / Supabase 기본 지원
- 대부분의 수업 자료 기준 버전
